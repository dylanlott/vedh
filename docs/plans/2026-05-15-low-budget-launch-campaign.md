# vEDH Low-Budget Launch Campaign

## Goal
Run a low-budget **open beta launch** for vEDH that attracts real Commander players, converts them into active game creators, and produces fast product feedback without spending heavily on ads.

## Recommended launch position
**Primary promise:**

> vEDH helps Commander players goldfish faster, track real board state, and review games after the fact.

This is a better launch angle than trying to sell vEDH as a broad generic game engine right now.

## 30-day launch targets
- 100-250 signups
- 25+ users who create at least 1 game
- 10+ returning users
- 5-10 strong feedback conversations

## Success metrics to track weekly
- waitlist signups
- invites sent
- accounts created
- games created
- returning users
- feedback submissions
- % of invitees who actually start a game

## Pre-launch must-fix items
These are the highest-leverage items to finish before driving broader traffic:

1. Fix the bug where the logged-in user cannot see their games.
2. Ensure goldfishing flows work cleanly.
3. Make auth/cookie handling reliable.
4. Ship a simple invite / signup page.
5. Make the mobile experience usable enough for real testing.

## Nice-to-have but not launch-blocking
- forgot password flow
- richer board-view polish
- historical game log drawer
- full game engine work
- smart contract work
- broader branding cleanup

## Channel strategy

### Owned channels
Focus here most heavily.
- simple signup / invite page
- email list
- lightweight feedback loop (Discord, Telegram, Tally, or Typeform)

### Rented channels
Use for attention, not as the core asset.
- X
- TikTok
- Reddit
- Show HN later, only if product stability is good enough

### Borrowed channels
Use small creators and community operators, not paid influencers.
- Commander creators
- brewing accounts
- Discord community mods
- deck tech / gameplay micro-creators

Offer early access, direct founder feedback, and recognition as early testers.

## Phased launch plan

### Phase 1 - Prep (5-7 days)
Ship the funnel and remove obvious friction.

Deliverables:
- signup / invite page
- onboarding email
- working game creation -> play -> revisit loop
- one demo clip
- one feedback form

### Phase 2 - Quiet beta push (7-10 days)
Bring in the first small cohort without a big public splash.

Actions:
- invite the first 20-40 users
- do manual outreach to friends and MTG contacts
- post soft launch content on X and TikTok
- ask each tester what they were trying to do, where it broke, and whether they would use it again next week

### Phase 3 - Public open beta push
Open it up once the core loop is holding.

Actions:
- announce open beta publicly
- post launch thread on X
- post 2-3 clips on TikTok
- post carefully on Reddit
- send the email announcement
- consider Show HN only if stability is high

- [ ] Fix the logged-in game list bug
- [ ] Verify game creation -> join -> play -> revisit flow
- [ ] Verify goldfishing works end to end
- [ ] Verify auth / cookie behavior on desktop and mobile

- [ ] Record one mobile board-state demo

- [ ] Invite 20 hand-picked users manually
- [ ] DM friends / Commander contacts / known testers

- [ ] Publish X launch thread

## Founder-facing daily execution checklist

Use this as the short version when actually operating the launch.

### Every morning
- [ ] Check signups, accounts created, games created, and returning users
- [ ] Check where new users are getting stuck
- [ ] Pick the **one** product fix most likely to improve activation today
- [ ] Pick the **one** distribution action most likely to get qualified players today

### Every afternoon
- [ ] Ship one concrete product improvement
- [ ] Reach out to 5-10 real players manually
- [ ] Post one piece of launch content or one progress update
- [ ] Reply to every inbound question, DM, or feedback note

### Every evening
- [ ] Record what shipped
- [ ] Record how many new signups actually started a game
- [ ] Note the top 1-3 friction points users hit
- [ ] Decide tomorrow's top product fix and top distribution action

### Rules for staying focused
- [ ] Do not spend the day on branding polish if activation is broken
- [ ] Do not buy ads before the create-game -> play -> return loop is healthy
- [ ] Do not chase broad awareness before the first 20-40 users are getting value
- [ ] Prefer direct outreach and user conversations over passive posting
- [ ] Optimize for active players and useful feedback, not vanity traffic

### Weekly checkpoint
At the end of each week, answer:
1. Are signups turning into created games?
2. Are created games turning into repeat usage?
3. What is the biggest activation blocker right now?
4. Which channel is bringing the best testers?
5. Is the product stable enough to widen the audience next week?
